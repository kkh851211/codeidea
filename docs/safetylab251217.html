Executive Summary

세이프티랩 기획서(v1.0) 검토 결과, 소프트웨어 개발 관점에서 6가지 설계 공백이 확인되었습니다.
코드아이디어는 이 공백을 사전에 정의하여 일정 지연·비용 증가·책임 소재 불명확 리스크를 최소화하고자 합니다.

제안 배경
  • 기획서는 운영 시나리오와 목적이 명확하게 정의된 우수한 기획서
  • 다만 소프트웨어 설계 상세도가 낮아 개발 착수 전 공백 정의 필요
  • 공백 미해결 시 개발 중 추가 요청 → 일정·비용 증가 리스크 발생
  • 코드아이디어가 선제적으로 해결 방안을 제안하여 리스크 최소화
핵심 설계 원칙 (기획서 준수)
  • AI는 출입 판단을 보조하며, 최종 판단은 반드시 관리자가 수행
  • 모든 AI 판단에 앞서 커프 혈압계를 통한 2차 검증 루트 항시 제공
  • 모델 캘리브레이션과 사용자 캘리브레이션 명확히 분리
  • 현장 운영자의 판단을 안전하게 보조하는 출입 보조 장치로 설계

Solution Design

각 공백 항목에 대한 코드아이디어의 구체적 해결 방안입니다.

① Quality Gate 필수화
  • 필수 모듈로 격상 — 옵션에서 필수로 전환, 기준값 수치 정의
  • 얼굴 ROI 안정성 — 프레임 내 얼굴 위치 이탈 허용 범위 설정
  • 최소 조도 기준 — 측정 가능한 최소 밝기 수준 (lux 단위) 정의
  • 움직임 허용 범위 — 측정 중 허용 가능한 머리 움직임 픽셀 기준
  • 신호 품질 점수 — rPPG 신호 최소 품질 기준값 (SNR 기반)
② 관리자 인터페이스 확장
  • 실시간 알림 — 이상 판정 시 화면 팝업 + 알림음 즉시 통보
  • 부재 시 처리 규칙 — N분 미응답 시 자동 처리 방식 사전 정의
  • 측정 이력 조회 — 날짜·사용자·결과 기준 검색 및 조회
  • 통계 리포트 — 일별/주별 이상 판정 건수, 재측정 발생률 현황
  • 상태 모니터링 — 카메라·센서·혈압계 실시간 작동 여부 확인
  • 기준값 설정 — 혈압 임계치를 관리자가 현장 맞춤 조정 가능
③ 캘리브레이션 수집 도구 개발
  • 모드 분리 — 실험 모드 / 운영 모드 전환 기능으로 데이터 혼입 방지
  • 동시 기록 — AI 측정값과 커프 측정값을 같은 시점에 자동 매칭 저장
  • 세션 관리 — 실험 대상자별 세션 구분 및 데이터 품질 자동 검증
  • 보정 파이프라인 — 수집 데이터 기반으로 AI 모델에 보정값 적용 흐름 설계
④ 데이터 저장 구조 설계
  • 데이터 구조 분리 — 익명 측정과 개인 식별 데이터를 별도 구조로 관리
  • 측정값 매칭 — AI 측정값과 커프 측정값을 동일 세션으로 연결
  • 개인정보 처리 — 수집 동의·보관 기간·삭제 시점 명시 및 마스킹 처리
  • 확장성 고려 — 대시보드·다중 키오스크 연동 대비 확장 가능한 구조
⑤ 사용자 UX 화면 설계
  • 안정화 유도 화면 — 측정 전 10~15초 "호흡을 고르세요" 안내 (혈압 정확도 향상)
  • 측정 결과 화면 — 수치 직접 표시 vs 정상/주의/위험 등급 표시 방식 결정
  • 재측정 안내 화면 — 재측정 이유를 사용자에게 명확히 전달
  • 휴식 대기 화면 — 5분 타이머 및 안내 메시지 표시
  • 동의·면책 화면 — 생체정보 수집 동의 및 "의료 진단 아님" 안내 문구
⑥ 시스템 예외 상황 대응
  • 네트워크 단절 — 오프라인 시 키오스크 동작 방식 사전 결정 (완전 차단 vs 로컬 판단)
  • 센서 오류 — 오류 감지 → 관리자 알림 → 수동 대응 절차 자동 연결
  • 혼잡 시간대 — 연속 측정 과부하 시 대기열 안내 및 응답 속도 유지
  • 시스템 로그 — 측정 실패·오류·관리자 개입 이력 별도 기록 (사후 분석용)

  • 캘리브레이션 수집 도구 없이는 AI 모델 보정 실험 시작 불가
  • 데이터 구조가 먼저 설계되어야 모든 모듈 개발 가능
  • 관리자 기능 범위 미확정 시 개발 중 반복 요청 불가피
2순위 모듈 — 개발 진행 중 병행 정의
  • UX 화면은 1순위 모듈 설계 후 구체화 가능
  • 예외 상황 대응은 기본 플로우 완성 후 추가 정의 가능
  • 단, 현장 안정성 테스트 전까지는 반드시 완료 필요

일정 관련 주의사항
  • 캘리브레이션 실험 대상자 확보가 일정에서 가장 큰 변수입니다. 수백 명 모집 계획을 세이프티랩이 사전에 준비해야 합니다.
  • 1단계 데이터 구조 설계가 지연되면 이후 모든 단계 일정이 연쇄 지연됩니다. 착수 전 최우선 확정이 필요합니다.
  • 관리자 기능 범위는 3단계 착수 전까지 세이프티랩과 최종 확정해야 합니다.
  • 5단계 현장 테스트는 세이프티랩이 실제 운영 환경(조명·공간)을 제공해야 의미 있는 검증이 가능합니다.
  • 총 15주는 제안 기준이며, 협의 후 조정 가능합니다.
Risk Management

프로젝트 진행 과정에서 발생 가능한 리스크와 코드아이디어의 대응 방안입니다.

리스크 목록 및 대응 방안
리스크 발생 가능 시점 대응 방안
Quality Gate 부재로 오탐 발생 및 신뢰도 하락 현장 운영 초기 Quality Gate 필수화 및 기준값 사전 수치 정의
캘리브레이션 도구 부재로 전체 일정 지연 캘리브레이션 실험 착수 시 수집 도구를 1단계 개발 범위에 포함
관리자 기능 추가 요청으로 개발 범위 확대 개발 중 ~ 운영 초기 관리자 기능 전체를 초기 설계에 포함
초기 데이터 구조 오류로 모델 개선용 데이터 활용 불가 모델 업데이트 시 1단계에서 확장성을 고려한 데이터 구조 설계서 작성
개인정보 처리 미비로 인한 법적 리스크 운영 중 언제든지 동의 화면 및 데이터 처리 방침 초기 설계에 반영
하드웨어 변경 시 소프트웨어 재작업 발생 유지보수 단계 HW-SW 인터페이스 명세를 계약서에 명시
최우선 대응 필요 리스크
  • Quality Gate 미필수화 — 현장 신뢰도에 직결. 초기 설계에서 반드시 필수 모듈로 확정 필요
  • 캘리브레이션 도구 부재 — 이 도구 없이는 AI 모델 보정 자체가 불가능. 개발 1단계에 즉시 포함
  • 데이터 구조 미설계 — 모든 모듈의 기반. 가장 먼저 확정해야 이후 개발이 가능
법적 리스크 별도 유의사항
  • 얼굴 인식·혈압·체온은 개인정보보호법상 민감 정보로 분류됨
  • 얼굴 인식 옵션 사용 시 별도 동의 절차 화면이 법적으로 필수
  • "본 측정값은 의료 진단이 아닙니다" 면책 문구를 키오스크 화면에 반드시 포함
  • 데이터 보관 기간 및 삭제 정책을 사전에 문서화하여 보관
